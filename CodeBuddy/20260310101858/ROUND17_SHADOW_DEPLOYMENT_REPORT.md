## 影子部署方案

### 部署模式
```
┌─────────────────────────────────────────────────────────────┐
│                    真实 Matrix 会议流                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   会议输入 ──┬──► 旧系统 (当前主系统) ──► 执行决策            │
│             │         │                                     │
│             │         └──► 决策结果 (实际执行)               │
│             │                                                │
│             └──► 影子系统 (只观察) ────► 建议决策             │
│                       │                                     │
│                       └──► 对比记录 ────► 审计日志           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 核心规则
1. **并行运行** - 影子系统实时处理所有会议
2. **只记不拦** - 影子建议仅记录，不影响实际执行
3. **旧主新影** - 旧系统保持唯一执行权
4. **差异审计** - 所有分歧完整记录

### 部署配置
```python
{
    "deliberation_threshold": 75.0,   # 协商评分门槛
    "review_threshold": 80.0,          # 审验评分门槛
    "max_defects": 1,                  # 最大缺陷容忍
    "support_weight": 1.0,             # 支持权重
    "oppose_penalty": 1.5,             # 反对惩罚
    "veto_penalty": 3.0                # 否决惩罚
}
```

## 5个核心观察指标

### 1. False-Block Rate in Real Meetings
- **定义**: 旧系统批准但影子系统拦截的比例
- **门槛**: ≤ 15%
- **意义**: 衡量影子系统是否过度严格

### 2. Review Disagreement Rate
- **定义**: 协商评分与审验评分差异 > 20 的比例
- **监控**: 趋势是否稳定下降
- **意义**: 评估第三方审验的有效性

### 3. Extra Round Overhead
- **定义**: 影子系统建议的额外轮次平均值
- **门槛**: ≤ 0.4 (即 40% 会议建议额外轮次)
- **意义**: 衡量系统对效率的影响

### 4. Human Override Rate
- **定义**: 人工覆盖影子建议的比例
- **监控**: 低且无集中抱怨
- **意义**: 评估影子建议的可接受度

### 5. Accepted-Risk Miss Rate ⭐ NEW
- **定义**: 旧系统放行、影子系统高置信拦截的案例中，后续验证确实暴露问题的比例
- **监控**: 30天后验证 Accepted-risk 案例
- **意义**: 区分"误拦"与"提前发现风险"

### 阶段划分

| 阶段 | 时间 | 目标 | 检查点 |

| 初期 | Day 1-7 | 稳定运行 | 系统无故障，日志正常 |

| 后期 | Day 22-30 | 评估准备 | 会议数 ≥ 50，指标计算 |

### 每日监控
- 影子系统运行状态
- 审计日志写入情况
- 新旧系统差异事件

### 30天判定
满足以下条件 → **Promote to Default**:
- False-Block Rate ≤ 15%
- Decision Alignment ≥ 75%
- Extra Rounds Ratio ≤ 40%
- 总会议数 ≥ 50
- 运行天数 ≥ 30

不满足 → **Keep Shadow Longer** 或 **Retune and Rerun**
